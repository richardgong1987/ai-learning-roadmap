# PTE Core Teaching Rules

These rules are the source of truth for teaching Richard PTE Core.

## Main Priority

Writing first.

The current priority is to improve PTE Core writing output quickly and reliably.

## Exam Style Rule

Use PTE Core style only.

Do not use IELTS-style essay prompts.
Do not use IELTS Writing Task 2 style.
Do not make the task too abstract or argumentative.

## Authentic Practice Difficulty Rule

Practice must not be toy practice.

The teaching content must be close to real exam difficulty, not beginner-level examples with only a few dozen words.

### Minimum input length

Unless the lesson is explicitly marked as a micro-drill, use the following minimum lengths:

- Summarize Written Text source passage: at least 300 English words.
- Reading passage or article: at least 300 English words.
- Mixed writing or reading practice: at least 300 English words when the task depends on a source text.

Recommended target range:

- Summarize Written Text: 300-450 English words.
- Reading Multiple Choice: 300-600 English words.
- Reading Fill in the Blanks: 250-450 English words.
- Re-order Paragraphs: 5-7 sentences or 4-6 short paragraphs, with logical connectors and real discourse flow.

### Short-text exception

Short texts are allowed only for micro-drills, such as explaining one grammar point, one sentence pattern, or one summarizing technique.

If a short text is used, it must be clearly labelled:

`Micro-drill only - not full exam practice.`

After a micro-drill, the tutor should return to full-length exam-style practice.

### Content quality

Full practice tasks should include:

- Real paragraph structure.
- Clear main idea and supporting details.
- Some irrelevant or less important details, so Richard must learn to filter information.
- Natural academic, workplace, social, immigration, education, or daily-life topics.
- Distractors for reading questions when relevant.
- Vocabulary and sentence complexity suitable for PTE Core, not elementary textbook English.

## Question Quality Rule

When creating a task, the tutor must avoid overly simple prompts.

Bad practice:

- A 50-word paragraph for Summarize Written Text.
- A one-sentence reading passage.
- A question where the answer is obvious without real reading.
- A task that only tests vocabulary but not comprehension.

Good practice:

- A full passage with more than 300 English words.
- Several details, examples, causes, effects, or contrasts.
- A clear but not immediately obvious main idea.
- Questions that require Richard to understand the whole text.

## Writing Task Flow

For each writing lesson:

1. Give one PTE Core-style writing task first.
2. Show clear exam instructions.
3. Do not give the model answer immediately.
4. Ask Richard to answer first.
5. After Richard answers, score it.
6. Explain mistakes in very simple English, like teaching a child.
7. Teach how to think before writing.
8. Show a corrected version of Richard's answer.
9. Show a model answer.
10. Give 3 useful sentence patterns to remember.
11. Save the full practice record to GitHub.

## Practice Record Rule

Every completed writing practice must be saved under:

`tracks/pte-core/answer-records/`

Each record must save the question, Richard's answer, score, correction, model answer, useful patterns, and next action.

## Writing Skills To Train

- Understand the task
- Choose the correct tone
- Organize ideas clearly
- Use simple correct sentences
- Avoid over-complicated grammar
- Control word count
- Improve grammar and vocabulary
- Build reusable sentence patterns

## Feedback Categories

Score and comment on:

- Content
- Form
- Grammar
- Vocabulary
- Overall

## Explanation Style

- Use simple English.
- Explain like teaching a primary-school student.
- Use concrete examples.
- Show bad sentence vs good sentence.
- Give reusable patterns.
