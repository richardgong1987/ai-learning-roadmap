# Convex Optimization Learning System

This file is the single entry point for the Convex Optimization learning track.

## Source of Truth

GitHub is the source of truth.

Do not rely on chat memory.
Do not restart from a different lesson unless `progress.md` says so.

## Course Goal

Develop a working knowledge of convex optimization: recognize, formulate, analyze, and solve convex optimization problems.

## Required Startup Procedure

Before every lesson, read these files in this order:

1. `tracks/convex-optimization/learning-system.md`
2. `tracks/convex-optimization/progress.md`
3. `tracks/convex-optimization/curriculum.md`
4. `tracks/convex-optimization/teaching-rules.md`

## Lesson Execution Workflow

1. Read the required startup files.
2. Identify the next lesson from `progress.md`.
3. Teach according to `teaching-rules.md`.
4. Complete the lesson.

## Definition of Lesson Completion

Teaching a lesson does not mean the lesson is completed.

A lesson is completed only when all required GitHub updates have been written and committed.

Required updates:

1. Update `tracks/convex-optimization/progress.md` to the next lesson.
2. Add a lesson note under `tracks/convex-optimization/lesson-notes/`.
3. Add or update a daily log under `tracks/convex-optimization/daily-log/`.
4. Commit all changes to GitHub.

If GitHub was not updated, the lesson is incomplete and should resume from the same lesson next time.

## Safety Against Forgetting

If there is a conflict between chat memory and GitHub, trust GitHub.

If the tutor is unsure where to continue, read `progress.md` again.

If the tutor is unsure how to teach, read `teaching-rules.md` again.
